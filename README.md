# CharOps - 字符操作工具

很久前刚学C++那会儿做的一个用于字符级位运算的 C++ 命令行工具。我记得当时觉得挺有意思的。现在回过头来，就把它重新上传留作纪念吧。该工具主要用于安全研究、CTF 比赛中的命令执行绕过场景，通过逆向位运算找出能够产生目标字符的字符组合。

## 工具介绍

在某些 PHP 命令执行场景中，需要绕过特殊字符过滤。本工具接受一个 payload 字符串，根据指定的工作模式对每个字符进行逆向位运算，找出满足条件的字符对 (j, k)。

### 应用场景

- **PHP 命令执行绕过** - 构造可绕过过滤的 payload
- **CTF 逆向工程** - 分析字符变换逻辑
- **安全研究** - 理解位运算对字符的影响

### 示例

例如 payload 为 `AJ`，模式为 XOR (0)：

```
A = '!' ^ '`'  →  输出 !*
A = ':' ^ '{'  →  输出 :{
A = '<' ^ '}'  →  输出 :}
...
J = '|' ^ '6'  →  输出 |6
J = '*' ^ '`'  →  输出 *`
J = '{' ^ '1'  →  输出 {1
```

因此输出可以是 `!*`、`:{`、`|6` 等任意满足条件的组合。

## 工具原理

对于 payload 中的每个字符 `payload[i]`，遍历 ASCII 码 33-126 的所有可打印字符，找到满足条件的字符对：

| 模式 | 运算公式 | 输出格式 |
|------|----------|----------|
| 0 (异或) | `ascii[j] ^ ascii[k] = payload[i]` | `s1^s2` |
| 1 (或) | `ascii[j] \| ascii[k] = payload[i]` | `s1\|s2` |
| 2 (与) | `ascii[j] & ascii[k] = payload[i]` | `s1&s2` |
| 3 (非) | `!ascii[j] = payload[i]` | URL 编码结果 |

### 输出说明

- **模式 0/1/2**：输出两个字符串，用运算符连接，如 `s1^s2`
- **模式 3**：输出单个字符串（按位取反结果）
- **找不到满足条件时**：输出 `Oops!`

### badchars 参数

使用 `-b` 指定坏字符，运算时需满足：`badchars` 中不包含结果中的任何字符。

## 功能特性

- **异或模式** - 字符异或运算
- **或模式** - 字符或运算
- **与模式** - 字符与运算
- **非模式** - 字符按位取反运算
- **URL 编码** - 可选的 URL 编码输出
- **坏字符过滤** - 排除指定字符

## 构建

```bash
cd build
CXX=g++ cmake ..
make
```

## 使用方法

```bash
./tools -p <payload> -m <mode> [-b <badchars>] [-u]
```

### 参数说明

| 参数 | 描述 | 必填 |
|------|------|------|
| `-p, --payload` | 输入字符串（大小写字母、数字、常用标点符号） | 是 |
| `-m, --mode` | 工作模式：0=异或, 1=或, 2=与, 3=非 | 是 |
| `-b, --badchars` | 坏字符（排除的字符） | 否 |
| `-u, --urlencode` | 输出前进行 URL 编码 | 否 |
| `-h, --help` | 打印帮助信息 | 否 |

### 使用示例

```bash
# 异或模式
./tools -p "AJ" -m 0

# 指定坏字符
./tools -m 0 -b '!*' -p AJ

# URL 编码输出
./tools -p "test" -m 0 -u
```

## 注意事项

- payload 只能包含：大小写字母、数字、编程中常用的标点符号
- 如果使用 `-b` 指定坏字符，运算时结果中不能包含这些字符
- 如果找不到满足条件的输出，程序会输出 `Oops!`
- 使用 `-u` 选项时，输出前会进行 URL 编码

## 技术要求

- 使用 CMake 组织工程
- 使用 CLI11 解析命令行参数
- C++17 标准
- 静态链接版本
